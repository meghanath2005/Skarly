Locked MVP Scope

Phase 1 freezes the first version so the build stays demo-worthy and achievable. The product is a creation flow, not a streaming app or remix platform.

Core loop

Record or upload

Capture up to 30 seconds of user singing, or select one short audio file from the phone.

Choice

Pick one genre

Support Lo-fi, Piano, Pop, Rock, Indian classical, EDM, Acoustic, and Cinematic.

Result

Play and download

Show dummy processing first, then present a final MP3 player, download, and share action.

No remixing

Existing copyrighted songs are outside version 1.

No voice cloning

The MVP uses the user's original vocal recording only.

No payments

The prototype should feel free, direct, and easy to try.

No social feed

History is private and utility-focused for now.

Creator Workspace

User identity is framed around creative storage and privacy, not a boring guest-versus-login split. Guests can create immediately; saved creators keep private history and defaults.

Guest Creator

Can record, generate, download, and keep temporary drafts for the current session.

Saved Creator

Keeps track history, renamed demos, saved defaults, and private workspace sync later.

Track States

Temporary, saved, downloaded, processing, failed, and retry states appear on every track row.

Voice Privacy

Private by default, delete raw recording, keep final mix only, and export/delete controls later.

Figma Build System

Recreate this board in Figma as named styles first, then design screens from reusable components. Treat it like a native iOS dark-mode app: SF system type, rounded Health-style cards, obvious actions, and restrained effects.

Color styles

Background/App Black, Surface/Card Charcoal, Surface/Elevated Gray, Text/Primary, Text/Secondary, Accent/Health Blue, Accent/Audio Purple, Accent/Record Pink.

Text styles

Use SF Pro defaults: Large Title 34/41 Bold, Title 24/30 Semibold, Headline 17/22 Semibold, Body 15/21 Regular, Caption 12/16 Medium.

Layout styles

Mobile frame 390 x 844, screen padding 20, card radius 8, component gap 12, section gap 24, bottom navigation height 64.

Effect styles

Use dark-mode depth sparingly: 0 18 44 rgba(0,0,0,.32). Avoid neon outer glows, blurred color blobs, and heavy gradient overlays.

Acceptance Checklist

Phase 1 and Phase 2 are complete when these are true.

  • MVP flow is fixed from start through result download.
  • All 12 screens have a defined purpose and main action.
  • Dummy frontend behavior is enough for Phase 3.
  • No copyrighted remixing is implied in version 1.
  • No AI singer cloning is implied in version 1.
  • Eight genres are visible and selectable.
  • Processing states support future backend job status.
  • Visual direction feels like a clean free music tool.
  • Components map to React Native Expo implementation.
  • Empty, loading, failed, and ready states are planned.
